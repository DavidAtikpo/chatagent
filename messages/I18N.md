# Internationalisation du site web

## Fichiers

| Fichier | Rôle |
|---------|------|
| `messages/fr.json` | **Source** — tous les textes en français |
| `messages/en.json` | Anglais (généré) |
| `messages/de.json` | Allemand (généré) |
| `messages/it.json` | Italien (généré) |
| `messages/es.json` | Espagnol (généré) |
| `messages/pt.json` | Portugais (généré) |

L'utilisateur voit des **fichiers JSON pré-traduits** chargés par `I18nProvider` — pas de traduction en direct dans le navigateur.

## Générer les traductions

Utilise **Google Translate** via `google-translate-api-x` (gratuit, sans clé).

```bash
cd web
npm run i18n:translate
```

**Par défaut = incrémental** : seules les clés **manquantes** ou encore **identiques au français** (ajoutées via `i18n:sync`) sont traduites. Le reste est conservé.

```bash
npm run i18n:translate -- --locale en          # anglais, nouvelles clés seulement
npm run i18n:translate -- --section home       # une section de fr.json
npm run i18n:translate -- --force              # tout retraduire depuis zéro
npm run i18n:translate -- --dry-run            # aperçu sans écriture
```

Workflow recommandé après avoir ajouté des clés dans `fr.json` :

```bash
npm run i18n:sync        # 1. copie la structure (nouvelles clés en FR)
npm run i18n:translate   # 2. traduit uniquement ce qui est encore en français
```

## Sans traduction (repli français)

Aligne la structure des JSON sur `fr.json` sans appeler Google :

```bash
npm run i18n:sync
```

Les clés déjà traduites sont conservées ; les nouvelles restent en français.

## Utilisation dans les composants

```tsx
"use client";
import { useT } from "@/i18n/context";

export function MyComponent() {
  const t = useT();
  return <h1>{t("dashboard.sites.title")}</h1>;
}
```

Avec variables :

```tsx
t("home.hero.subtitle", { saasName: "ChatAgent" })
```

## Workflow

1. Ajouter les chaînes dans `fr.json`
2. Lancer `npm run i18n:translate` (ou `i18n:sync` en attendant)
3. Les pages utilisent `t("clé")`
