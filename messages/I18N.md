# Internationalisation du site web

## Fichiers

| Fichier | Rôle |
|---------|------|
| `messages/fr.json` | **Source** — tous les textes en français |
| `messages/en.json` | Anglais (généré) |
| `messages/de.json` | Allemand (généré) |
| `messages/it.json` | Italien (généré) |
| `messages/es.json` | Espagnol (généré) |
| `messages/pt.json` | Portugais (généré) |

## Générer les traductions

Prérequis : `ANTHROPIC_API_KEY` dans `api/.env`

```bash
cd web
npm run i18n:translate
```

Une seule langue :

```bash
npm run i18n:translate -- --locale en
```

Test sans appel API :

```bash
npm run i18n:translate -- --dry-run
```

## Utilisation dans les composants

```tsx
"use client";
import { useT } from "@/i18n/context";

export function MyComponent() {
  const t = useT();
  return <h1>{t("dashboard.sites.title")}</h1>;
}
```

Avec variables :

```tsx
t("home.hero.subtitle", { saasName: "ChatAgent" })
```

## Sélecteur de langue

```tsx
import { LanguageSwitcher } from "@/components/language-switcher";
```

La langue est stockée dans le cookie `site_locale`.

## Prochaine étape

Remplacer progressivement les textes en dur dans chaque page par `t("clé")`.
Le fichier `fr.json` est la référence : toute nouvelle chaîne UI doit y être ajoutée d'abord, puis relancer `npm run i18n:translate`.
